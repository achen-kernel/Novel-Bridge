# Harness 习惯

把 harness engineering 作为实用的 vibe coding 习惯来训练，不作为理论课。

## 新手习惯

每个阶段强化：

- 开发前定义阶段边界
- 说明不做什么
- 让 Agent 说明预计修改哪些文件
- 接受“完成”前要求证据
- API 或 DTO/VO 变化后更新字段溯源
- 把 Agent 偏差记录成下次提示规则
- 把上下文紧凑地持久化到项目文档

## 个人工作法

在版本检查点、重复错误或项目结束时更新 `personal-vibecoding-playbook.md`。

使用有证据的规则：

```text
规则：修改后端 API 时，同时检查前端 API 调用。
证据：v1-auth 阶段曾因 auth.ts 仍使用旧字段导致失败。
置信度：high
适用场景：接口路径、请求体、响应体、DTO 或 VO 变化时。
```

## 需求漂移处理

把新请求分类：

- `keep`：属于当前阶段。
- `defer`：有价值，但放到后续阶段。
- `reject`：过重或不适合当前新手项目。

始终给出学习者还能构建并理解的最小下一步。
