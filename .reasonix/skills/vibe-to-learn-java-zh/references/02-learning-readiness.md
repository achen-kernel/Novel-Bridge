# 学习准备检查

在把生成的项目 Plan 转成学习计划前使用。

## 目标

判断这个 Plan 是否已经清楚到足够让新手边做边学。这是轻量门槛，不是完整设计评审。

## 检查项

- 项目目标能用一两句话说清楚。
- MVP 有 3-5 个核心功能，或可以缩小到这个范围。
- 每个功能能对应页面、API、Service、数据和验收方式。
- 技术栈没有对新手明显过重。
- 每个阶段都能产生可运行证据。
- 每个阶段有 1-3 个学习目标。
- 高风险功能应延后：支付、多租户权限、消息队列、推荐系统、复杂权限、微服务、高级部署。

## 输出

只使用三种状态：

- `Green`：可以开始。
- `Yellow`：缩小范围后开始。
- `Red`：当前不适合新手，先重新切 MVP。

始终给出最小可继续版本。

## 足够好标准

不要因为缺少企业级文档而阻塞。只要 Agent 能创建阶段计划，且学习者能说明下一阶段要做什么、怎么验收、练什么，就可以开始。
