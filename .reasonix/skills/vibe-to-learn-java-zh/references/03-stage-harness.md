# 阶段 Harness

同一时间只保留一个活跃阶段。

## 阶段状态

允许的状态：

- `planned`
- `in_progress`
- `review`
- `closed`
- `snapshot`

当前阶段未关闭或未明确暂停前，不要启动新阶段。

## Current Stage 字段

保持 `current-stage.md` 紧凑：

- 阶段 id 和版本
- 状态
- 功能目标
- 不做事项
- 可能修改文件
- 学习目标
- 验收证据
- 后端/全栈检查点
- 练习候选
- 风险

## Agent 契约

阶段开发前说明：

- 本阶段做什么
- 本阶段不做什么
- 可能涉及哪些文件或模块
- 必须怎么验证
- 什么时候更新 `flow-map.md`
- 什么时候添加练习标记

## 关闭条件

阶段关闭必须具备：

- 功能证据：构建、测试、API、页面或手动验收
- 学习证据：链路或阶段说明已更新
- 练习证据：练习候选已评审，或明确跳过
- 复盘证据：如果发生 Bug、偏差或决策变化，已记录

缺少证据时，状态设为 `review`，不要设为 `closed`。
